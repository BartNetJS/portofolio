System Architecture

The integration platform operates as a Windows Service on the DeltaV Application Station, acting as a bridge between:

  • DeltaV Control System - Emerson DeltaV via OPC DA for tag reading/writing
  • Local Database - SQL Server for transaction management and caching
  • Backend Database - Enterprise backend via SQL Linked Server for orders and recipes

Data Flow Categories

  • LOP (Loading Operations) - Badge validation, order lookup, loading control parameters
  • SFC (Shop Floor Control) - Loading station status, real-time progress updates
  • TNK (Tank Operations) - Tank inventory, connection status, density values
  • Recipe Management - Product recipes, blending percentages, tank mappings

üîÑ Badge Authentication Flow

Complete Badge-to-Loading Sequence

The badge authentication process involves multiple systems working together in real-time:

  1. Badge Scan - Driver scans badge at loading station ‚Üí DeltaV reads CSN (hex)
  2. CSN Conversion - Interface converts CSN to Badge ID via backend lookup
  3. Order Validation - Backend validates badge and returns order details
  4. Local Transaction - Create loading transaction with unique ID
  5. Recipe Loading - Write product recipe and tank parameters to DeltaV
  6. Loading Process - Monitor progress, update quantities in real-time
  7. Completion - Report final quantities to backend, archive transaction

Status Handling

  • Normal Flow - Badge validated, loading proceeds automatically
  • Already Loaded - Badge rejected if already used
  • Wrong Station - Badge assigned to different loading bay
  • No Recipe - Product not configured for loading
  • Interrupted - Loading can be resumed with same badge
  • Offline Mode - Operator creates manual badge when backend unavailable

üì° Integration Points

DeltaV OPC Tags

The interface manages hundreds of OPC tags across multiple categories:

  • Badge Tags - DV_BADGE_CODE, MIS_BADGE_ID, BADGE_STATUS
  • Loading Tags - PRODUCT, ORDER_NUMBER, MT_ASKED, MT_DELIVERED
  • Recipe Tags - TANK_1..4, PERCENTAGE_1..4, DENSITY values
  • Status Tags - LS_STATUS, LOAD_START, LOAD_STOP, ERROR_MSG
  • Watchdog - RAMP counter for health monitoring

Backend Stored Procedures

  • Badge Operations - Badge_Convert_CSN, Badge_Read_New, Badge_Loaded_New
  • Recipe Sync - GetRecipes, GetRecipe_Parameters, CheckRecipesStatus
  • Status Updates - Loading_Stations_Status, Tank_Prod_Densities
